SQL Visualizer

A self-contained, browser-based tool for learning and exploring SQL concepts through interactive diagrams, live queries, and step-by-step visual breakdowns.

NO SERVER, NO DEPENDENCIES, NO SETUP REQUIRED. Get Started.

What It Is

SQL Visualizer transforms abstract SQL concepts into clear, interactive visual representations. Each topic is accompanied by practical, working examples, allowing users to see how queries behave in real scenarios. Data connection diagrams update dynamically as join types or relationships are modified, providing immediate visual feedback on table interactions. Execution phases can be expanded with a single click, revealing detailed, step-by-step insights into how queries are processed. Additionally, an in-memory dataset displays query results instantly, enabling users to experiment freely and understand outcomes without delay.

This combination of visualization, interactivity, and real-time feedback makes SQL Visualizer a powerful tool for learning, teaching, and exploring SQL.

Why I Built It

Most SQL learning resources explain concepts through static text and diagrams. That works up to a point, but the moment you try to write a window function or debug why a WHERE clause can't reference a SELECT alias, static explanations fall short. You need to see the data moving.

I built SQL Visualizer to bridge that gap — a reference tool that lives alongside your editor, answers "what does this actually return?" immediately, and makes the mental models concrete rather than theoretical. It's aimed at developers who already know some SQL but want a faster way to reason through the trickier parts.

Design constraint: everything had to work offline, in a single file, with zero dependencies. The SQL engine, the in-memory database, and all interactivity are hand-rolled in vanilla JavaScript.

Topics Covered

SQL Joins
Interactive

Covers all six join types — INNER, LEFT, RIGHT, FULL OUTER, CROSS, and SELF. Each has a live Venn diagram, a result preview drawn from real sample data, and annotated syntax. Common pitfalls like implicit Cartesian products are called out explicitly.

Execution Order
Visual

Execution order covers an 12-steped extended version of the basic eight-step logical execution processing phases: FROM → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT. Each step explains what the engine does at that phase and why it matters for writing correct queries.
